# 文献知识（按需求/总体/分项/技术域摘录）

## 一、需求维度

### 组织架构管理需求
- GB/T 22239-2019 要求三级等保系统须具备统一的用户身份管理机制
- 主流方案支持10级组织树，大型项目可达15级
- 组织变动历史追溯是审计合规的刚性需求

### 身份认证需求
- OAuth 2.0 (RFC 6749) 授权码模式是B端SSO的主流选择
- OpenID Connect (OIDC) 在OAuth2之上提供身份层，适合统一认证
- SAML 2.0 在政府/央企系统中仍广泛使用，需兼容
- CAS协议是Java生态的传统SSO方案，存量系统适配成本低
- 多因子认证已成为等保三级的基本要求
- 国密SM2/SM3/SM4在党政机关是强制要求（GM/T 0054-2018）

### 权限管理需求
- RBAC是当前最成熟的企业级权限模型（NIST标准）
- ABAC在动态权限场景（如数据纬度、时间纬度）优于RBAC
- 角色互斥（SoD）是审计合规的必备能力

### 性能需求
- Keycloak官方数据：单节点支持约100万用户，500+并发认证
- 缓存层Redis在认证场景下可将响应时间从DB级的50ms降至1-2ms
- Session分布式存储是SSO高可用的关键

## 二、总体维度

### 架构模式
- 微服务架构是当前统一认证平台的业界标准架构
- 认证服务+授权服务+用户服务+审计服务 四模块分离
- API Gateway作为统一入口（Spring Cloud Gateway/Kong）
- 前后端分离，前端无状态，Token承载认证信息

### 技术栈选型
- Java Spring Cloud 在企业级市场占有率最高（约65%）
- .NET Core 在微软生态中占优
- Go在性能敏感场景（网关、Token校验）有优势
- 数据库：MySQL/PostgreSQL + Redis缓存
- 国产化：麒麟OS + 达梦DB + 东方通TongWeb

### 部署方案
- 容器化是标准部署方式，K8s编排
- 各模块独立扩缩容，认证模块可水平扩展
- 建议最小3节点集群保障99.9%可用性

## 三、分项维度

### 用户管理中心
- HR同步：SCIM 2.0协议是跨系统用户同步的国际标准
- 批量导入：支持Excel/CSV/LDIF多种格式
- 密码策略：bcrypt加盐哈希是业界最佳实践，计算开销≈100ms/次
- 自助服务：短信验证码对接（阿里云/腾讯云SMS网关）

### 身份认证中心
- JWT Token：RS256签名，短时效（15-30分钟）+ Refresh Token轮换
- Session管理：Redis集中存储，支持分布式部署
- TOTP：RFC 6238标准，30秒有效期，HmacSHA1算法
- LDAP/AD：Spring Security LDAP模块成熟对接方案
- 生物特征：预留接口标准（FIDO2/WebAuthn）

### 权限管理中心
- Spring Security + Method Security 是Java生态标准实现
- 数据权限：MyBatis-Plus/Mate-DataAuth 提供行级拦截方案
- 角色继承：树形角色结构，子角色继承父角色权限

### 安全审计中心
- ELK/EFK是日志审计的通用方案
- 日志不可篡改：区块链哈希链/数字签名方案
- 异常告警：设定阈值规则（5分钟内失败5次→锁定账号+告警）

## 四、技术域维度

### 密码学
- bcrypt：自适应哈希，计算强度可配（当前推荐≥10轮）
- 国密SM2：非对称加密，等效RSA 2048
- SM3：密码杂凑，等效SHA-256
- SM4：对称加密，等效AES-128
- TLS 1.3：当前最新安全传输协议

### 容器化
- Docker镜像：基于OpenJDK 17 Alpine，镜像控制在200MB以内
- K8s部署：HPA自动扩缩容，Ingress暴露CA服务
- ConfigMap/Secret管理配置和密钥

### 国产化适配
- 麒麟V10 + 达梦DM8是当前国产化主流组合
- Spring Boot在麒麟OS上已验证兼容
- JDBC驱动层适配达梦（DM8兼容Oracle模式）
