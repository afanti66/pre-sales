# 某单位统一用户管理与身份认证系统建设项目

## 一、项目背景
某单位现有多个业务系统（OA、财务、人事、档案、项目管理），各系统独立维护用户账号和权限体系，存在以下问题：
- 用户需记忆多套账号密码，经常忘记密码导致运维压力大
- 人员变动时无法在统一入口完成账号禁用/删除，存在安全隐患
- 各系统权限模型不一致，无法实现统一的访问控制策略
- 审计日志分散在各系统，无法实现全局安全审计
- 新业务系统上线需重新开发用户管理模块，重复造轮子

## 二、建设目标
建设一套统一的用户管理与身份认证平台，实现：
1. 统一用户管理：组织架构、用户账号、角色权限的集中管理和同步
2. 统一身份认证：单点登录（SSO），支持多种认证方式
3. 统一权限管理：基于RBAC/ABAC的细粒度权限控制
4. 统一安全审计：全平台操作日志的集中采集、存储和分析
5. 统一应用接入：提供标准化接口，新系统快速接入

## 三、功性能要求

### 3.1 功能要求

**F1 组织架构管理**
- 支持树形组织架构维护，至少10级深度
- 支持组织的新增、修改、合并、撤销
- 支持组织与人员的关联管理
- 支持组织变动历史追溯

**F2 用户生命周期管理**
- 用户注册、审核、激活、禁用、删除全生命周期管理
- 支持批量导入/导出（Excel/CSV）
- 支持与HR系统自动同步用户数据
- 用户密码策略管理（复杂度、有效期、历史次数）
- 自助密码重置功能（短信/邮箱验证码）

**F3 身份认证**
- 支持用户名/密码认证
- 支持短信验证码认证（对接短信网关）
- 支持动态令牌（TOTP/HOTP）
- 支持数字证书认证（国密SM2）
- 支持LDAP/AD域认证对接
- 支持OAuth2.0/SAML2.0/OIDC标准协议
- 支持CAS单点登录协议
- 支持生物特征认证（指纹/人脸，预留接口）

**F4 权限管理**
- 基于RBAC（基于角色的访问控制）
- 支持ABAC（基于属性的访问控制）扩展
- 角色继承与角色互斥
- 数据权限粒度可达到行级和字段级
- 支持按组织维度授权
- 支持临时授权和紧急通道

**F5 审计日志**
- 所有认证和授权操作全量记录
- 日志不可篡改，支持数字签名
- 日志存储至少保留6个月
- 支持日志实时检索和导出
- 支持异常行为告警（多次登录失败、异常时间访问）

**F6 应用接入管理**
- 提供标准REST API接口
- 提供应用注册和管理界面
- 支持应用密钥轮换
- 应用访问统计和监控

### 3.2 性能要求

| 指标 | 要求 |
|------|------|
| 用户规模 | 支持≥50,000注册用户 |
| 并发认证 | ≥500 TPS |
| SSO登录响应 | ≤2秒（95%分位）|
| 系统可用性 | ≥99.9% |
| 组织架构同步 | ≤5分钟（全量）|
| 审计日志写入 | ≥1000条/秒 |

### 3.3 安全要求

- 密码传输必须加密（HTTPS+TLS 1.2以上）
- 密码存储必须加盐哈希（bcrypt/国密SM3）
- 支持国密算法（SM2/SM3/SM4）
- 会话超时管理（空闲超时、绝对超时）
- 防暴力破解机制
- IP黑白名单
- 三级等保合规

## 四、技术架构要求

- 采用微服务架构，各模块可独立部署和扩展
- 支持容器化部署（Docker/Kubernetes）
- 数据库支持MySQL/PostgreSQL，可扩展
- 缓存层使用Redis
- 前端使用Vue.js或React
- 后端Java技术栈优先（Spring Boot/Cloud）
- 支持国产化环境适配（麒麟OS、达梦数据库）
