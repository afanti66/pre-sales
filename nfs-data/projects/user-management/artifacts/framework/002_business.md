## 三、业务原理

### 3.1 统一认证原理
一次登录，全网通行。通过建立全局认证会话（Session/Token），用户在任何一个接入应用完成认证后，访问其他应用无需重复输入凭证。

### 3.2 RBAC授权原理
访问权限不直接授予用户，而是通过角色作为中间层。用户→角色→权限的三层模型，实现权限管理的灵活性和可审计性。

### 3.3 单点登录原理
用户在认证中心完成登录后，生成全局票据（Ticket/Session）。各业务系统通过验证此票据，确认用户身份并建立本地会话，无需用户再次输入凭证。

### 3.4 审计追溯原理
所有认证和授权操作生成不可篡改的审计日志，日志链通过哈希算法串联，确保任何历史记录的修改都可被检测。

## 四、业务流程

### 4.1 用户注册→审核→激活流程
```
用户提交注册信息
  → 系统校验格式（密码复杂度、手机号格式等）
  → 通知管理员审核
  → 管理员审核通过/驳回
  → 审核通过→激活账号→发送通知给用户
  → 用户首次登录→强制修改初始密码→进入系统
```

### 4.2 SSO单点登录流程
```
用户访问业务系统A（未登录）
  → 业务系统A检测无Token
  → 重定向至认证中心
  → 认证中心展示登录页面
  → 用户输入账号密码（或短信码/TOTP等）
  → 认证服务验证凭证
  → 验证通过→创建全局Session
  → 生成授权码→回调业务系统A
  → 业务系统A用授权码换取AccessToken
  → 用户登录成功
  → 用户再访问业务系统B
  → 检测到全局Session→自动登录
```

### 4.3 权限判定流程
```
用户请求访问资源
  → 权限拦截器拦截请求
  → 提取用户ID和角色信息（从Token/缓存）
  → 查询角色-权限映射
  → 判定是否有该资源的访问权限
  → 有权限→放行；无权限→返回403
  → 操作结果写入审计日志
```
