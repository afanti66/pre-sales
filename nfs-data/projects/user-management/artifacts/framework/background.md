# 背景知识（框架阶段产出 · 供设计阶段使用）

> 本文档由框架智能体汇总：需求矩阵 + 业务级框架 + 系统级框架

## 一、项目概要

- 项目名称：某单位统一用户管理与身份认证系统
- 用户规模：≥50,000注册用户
- 并发认证：≥500 TPS
- 技术栈：Java Spring Cloud微服务，MySQL/达梦，Redis，ES
- 部署方式：容器化K8s，支持国产化（麒麟OS+达梦+东方通）

## 二、需求矩阵摘要

### F1 组织架构管理
- 10级组织树、合并拆分追溯、人员-组织关联
- **技术选型**: parent_id+level+path方案，MySQL递归查询
- **验收**: 增删改查≤1s

### F2 用户生命周期管理
- 注册/审核/激活/禁用/删除，HR同步(SCIM 2.0)，批量导入(Excel/CSV)，密码策略(bcrypt)
- **技术选型**: 状态机设计，SCIM协议客户端，EasyExcel
- **验收**: 1万用户导入≤30s，同步≤5min

### F3 身份认证
- 密码/短信验证码/TOTP/数字证书SM2/LDAP-AD
- 协议支持：OAuth2/OIDC/SAML2.0/CAS
- 预留：生物特征(FIDO2/WebAuthn)
- **技术选型**: Spring Security OAuth2 Client + nimbus-jose-jwt
- **验收**: 认证≤500ms，SSO登录≤2s

### F4 权限管理
- RBAC+ABAC，角色继承/互斥，数据权限行级
- **技术选型**: Spring Security Method Security + MyBatis拦截器
- **验收**: 权限判定≤100ms

### F5 审计日志
- 全量记录、哈希链防篡改、ES存储、异常告警
- **技术选型**: AOP+MQ+ES，保留6个月
- **验收**: 检索≤3s，写入≥1000条/秒

### F6 应用接入
- REST API、应用注册管理、密钥轮换
- **技术选型**: Spring Boot RESTful
- **验收**: 接口响应≤200ms

## 三、业务框架摘要

| 业务域 | 关键业务活动 | 业务类型 |
|--------|------------|---------|
| 用户管理域 | 组织/用户/同步 | 管理类+自动化 |
| 身份认证域 | 多因子认证/SSO/令牌管理 | 核心业务类 |
| 权限管理域 | 角色管理/授权判定/数据权限 | 核心业务类+管理类 |
| 安全审计域 | 日志采集/检索/告警 | 自动化类+管理类 |
| 系统管理域 | 应用接入/配置/监控 | 管理类 |

**核心业务流程**: 用户注册→审核→激活 → SSO登录 → 权限判定 → 审计记录

## 四、系统框架摘要

| 分系统 | 核心模块 | 关键接口 |
|--------|---------|---------|
| 用户管理服务 | 组织/用户/同步 | REST + SCIM 2.0 |
| 身份认证服务 | 认证/SSO/令牌 | OAuth2/OIDC/SAML/CAS |
| 权限管理服务 | 角色/授权/数据权限 | REST |
| 安全审计服务 | 日志采集/存储/告警 | REST + MQ |
| API网关 | 路由/限流/鉴权 | — |
| 前端管理台 | 管理界面/审计台 | — |

**通信方式**: REST同步 + RabbitMQ异步 + Nacos服务发现
**认证方式**: JWT(RS256, 15min) + RefreshToken(Redis, 7天)
