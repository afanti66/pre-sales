## 二、工作原理

### 2.1 微服务通信原理
各分系统间通过REST API同步通信，通过RabbitMQ异步解耦。API网关作为统一入口，负责认证拦截和请求路由。服务发现采用Nacos，配置管理采用Nacos Config。

### 2.2 认证工作原理
```
用户提交凭证 → 认证模块验证 → 验证通过 → 生成JWT
  → JWT含用户ID/角色/有效期 → 返回给客户端
  → 客户端携带JWT访问资源 → API网关校验JWT签名
  → 通过→路由到目标服务 → 目标服务解析JWT获取用户信息
```

### 2.3 Token工作原理
- AccessToken：JWT格式，RS256签名，有效期15分钟
- RefreshToken：Opaque格式（不透明令牌），Redis存储，有效期7天
- AccessToken过期→用RefreshToken换取新AccessToken
- RefreshToken过期→重新登录

### 2.4 权限判定工作原理
```
请求到达 → 权限拦截器 → 从JWT提取角色列表
  → 查询Redis缓存（角色→权限映射）
  → 缓存命中→直接判定
  → 缓存未命中→查询数据库→写入Redis
  → 比对该请求的资源+操作是否在权限集中
  → 通过/拒绝
```

### 2.5 审计日志工作原理
```
关键操作 → AOP切面捕获 → 封装审计事件对象
  → 发送到RabbitMQ → 审计服务消费
  → 计算SM3哈希（含上条日志哈希）→ 写入ES
  → 检查是否匹配告警规则 → 匹配→发送告警通知
```

## 三、工作流程

### 3.1 用户登录认证工作流
```
  [客户端]                [API网关]              [认证服务]            [Redis/DB]
     │                       │                      │                   │
     │──登录请求─────────────→│──路由→认证──────────→│                   │
     │                       │                      │──查询用户信息─────→│
     │                       │                      │←返回用户+密码哈希──│
     │                       │                      │──bcrypt比对───────│
     │                       │                      │──生成JWT─────────→│
     │                       │                      │←返回Token──────────│
     │←──Token───────────────│←──响应───────────────│                   │
     │                       │                      │                   │
```

### 3.2 SSO跨应用访问工作流
```
[用户]  → 访问应用A → 重定向到认证中心
  → 输入凭证 → 认证中心验证 → 生成全局Session
  → 回调应用A → 应用A获取Token
  → 用户访问应用B → 应用B检测到无Token
  → 重定向到认证中心 → 检测到全局Session
  → 自动生成授权码 → 回调应用B
  → 应用B获取Token → 用户自动登录
```

### 3.3 权限变更工作流
```
[管理员] → 选择角色 → 修改权限 → 提交变更
  → 系统校验（互斥角色检查、SoD检查）
  → 通过→更新数据库角色-权限映射
  → 清除Redis缓存中该角色的权限缓存
  → 写入审计日志 → 变更生效
```

## 四、系统接口

| 接口名称 | 协议 | 请求方式 | 用途 |
|---------|------|---------|------|
| 用户注册接口 | REST | POST /api/v1/users/register | 自助注册 |
| 用户查询接口 | REST | GET /api/v1/users/{id} | 获取用户信息 |
| 组织同步接口 | SCIM 2.0 | POST /scim/v2/Users | HR系统同步 |
| 认证接口 | REST | POST /api/v1/auth/login | 用户登录认证 |
| Token刷新接口 | REST | POST /api/v1/auth/refresh | 刷新AccessToken |
| Token校验接口 | REST | POST /api/v1/auth/validate | 资源服务校验Token |
| SSO授权接口 | OAuth2 | GET /oauth2/authorize | SSO授权码发放 |
| Token换发接口 | OAuth2 | POST /oauth2/token | 授权码换Token |
| 权限判定接口 | REST | POST /api/v1/auth/check | 资源权限判定 |
| 角色创建接口 | REST | POST /api/v1/roles | 角色管理 |
| 日志查询接口 | REST | GET /api/v1/audit/logs | 审计日志检索 |
| 应用注册接口 | REST | POST /api/v1/apps | 应用接入 |
