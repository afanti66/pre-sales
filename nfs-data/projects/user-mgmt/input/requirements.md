# XX集团统一用户管理系统项目 招标要求

## 一、项目背景
XX集团现有员工12000人，下属8个子公司，分布在全国12个城市。目前各子公司使用独立的用户管理系统。

痛点：
1. 信息孤岛——同一员工多系统多账号，无法统一管理
2. 权限混乱——权限模型不统一，审计困难
3. 认证薄弱——仅密码认证，安全隐患大
4. 入职离职低效——开账号平均3工作日，离职清理滞后
5. 合规压力——等保2.0要求统一身份认证和审计

## 二、总体要求
建设统一用户管理系统，实现五大统一：
- 统一身份管理（用户全生命周期）
- 统一认证管理（MFA+SSO）
- 统一权限管理（RBAC）
- 统一审计管理（日志+合规报表）
- 统一门户（自助服务台）

## 三、详细功能需求

### 3.1 身份管理
- 组织架构同步（AD/LDAP/HR系统）
- 账号全生命周期（创建/变更/冻结/注销）
- 用户自助服务（密码修改/权限申请/账号解锁）

### 3.2 认证管理
- 多因素认证（密码+短信/邮箱/USBKey）
- 单点登录SSO（OAuth2.0/SAML2.0/CAS）
- 生物特征认证（指纹/人脸 可选）

### 3.3 权限管理
- RBAC权限模型（角色/权限/继承）
- 功能权限+数据权限分离
- 权限申请审批流程（申请→审批→复核→生效）

### 3.4 审计管理
- 全量操作日志
- 登录审计（成功/失败+异常预警）
- 等保2.0合规报表
- 会话管理（在线监控/强制下线）

## 四、非功能需求
- 性能：10万用户在线，并发登录≥500TPS
- 可用性：99.99%（年停机≤52分钟）
- 安全：TLS1.3传输加密，bcrypt密码存储，防暴力破解
- 扩展：横向扩展，模块化设计
- 兼容：Chrome/Firefox/Edge + 移动端H5
